# Changelog

## 0.1.4 - 2026-03-31

- split the input editor into deeper button-style sections with dedicated `Field` and `Adorn` surfaces
- wired prefix and suffix content, clear action, password reveal, description text, and success messaging through live preview and exported React
- kept the new field-state composition preview/export honest while preserving clean lint, typecheck, and build verification

## 0.1.3 - 2026-03-31

- expanded the input state model and editor surface with additional production HTML attributes such as `id`, `title`, `tabIndex`, `dir`, `lang`, `inputMode`, `enterKeyHint`, `autoCapitalize`, `autoCorrect`, and numeric `min` / `max` / `step`
- kept live preview and exported React output aligned with the broader attribute surface

## 0.1.2 - 2026-03-31

- expanded the preset system from starter packs into larger generated button-style preset catalogs
- aligned the preset browser and preview reset flow more closely with the button-component product model
- preserved clean lint, typecheck, and production build verification after the canon sync

## 0.1.1 - 2026-03-31

- added a searchable preset catalog with full-state preset apply flow
- aligned the input studio more closely with the button-style UI Foundry editing model
- preserved clean lint, typecheck, and production build verification after the preset rollout

## 0.1.0 - 2026-03-30

- stabilized the standalone input app so build, lint, and typecheck pass
- synced shared layout and control primitives to the UI Foundry standard
- preserved a self-contained input playground and export workflow
